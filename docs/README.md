## üìù Documentation Standards

### When to Update Documentation

- **Goals**: When project vision or objectives change
- **Rules**: When adding new coding standards or architectural patterns
- **Workflows**: When development processes change or new workflows are added
- **Business Rules**: When calculation formulas or tax rates change

### How to Update Documentation

1. Edit the relevant markdown file
2. Maintain consistent formatting and structure
3. Add examples where helpful
4. Update the "Last Updated" date if present
5. Commit with descriptive message: `docs: update [topic]`

### Documentation Style

- **Clear and Concise**: Use simple language
- **Well-Structured**: Use headings, lists, and tables
- **Examples**: Include code examples and calculations
- **Links**: Cross-reference related documentation
- **Up-to-Date**: Keep information current
